# 基于知识图谱的"晋商精神"与平遥商号历史探索系统

## 技术栈

- **框架**: React 18 + Vite 5
- **路由**: React Router v6（多页面导航）
- **图谱可视化**: vis-network
- **数据**: 模拟JSON数据（后续可对接API）
- **样式**: CSS Modules 或全局 CSS，**整体视觉为水彩古风**（见下节）

## 视觉风格：水彩古风

全站统一采用**水彩古风**审美，便于 AI 与开发在实现时有一致标准。

- **色彩**：以淡彩为主——浅青、淡赭、藕荷、米白、宣纸色、留白；可叠加轻柔的晕染与渐变，避免高饱和、纯黑块。重点处可用略深的赭石、朱砂或青灰做点缀。
- **质感**：水彩的透明感、色块晕染、边缘柔和；可选浅色水痕、淡彩叠色、轻纹理背景（如宣纸/绢纹），整体轻盈有呼吸感。
- **字体**：中文优先宋体、楷体、仿宋或思源宋体等古典字体；标题可略加重，正文保持易读。
- **意象**：山水淡彩、云气、远山、古镇轮廓、草木晕染等，与晋商/平遥主题契合；线条可柔可细，不必拘泥于硬墨线。
- **组件**：导航栏、卡片、图谱节点与边、时间轴、输入框等采用淡彩描边或浅色块、柔和圆角、适量留白，整体温润不刺眼。
- **动效**：滚动视差与过渡以淡入淡出、轻微位移为主，保持舒缓。

以下各阶段涉及 UI 的部分均在此风格下实现。

## 首页滚动引入效果说明（供 AI 复现）

以下描述供 AI 阅读并生成**类似的简单滚动视差效果**，无需 GSAP，用原生滚动 + CSS transform 即可。

### 效果目标

- **长滚动段**：页面有一段「加长」的滚动距离（例如占 300vh～400vh），用户向下滚动时感觉在「穿过」同一块视觉区域。
- **固定视口**：该区域在视觉上铺满视口且**不随滚动移动**（内部用 `position: fixed` 的容器实现）。
- **多层视差**：容器内有多层元素（背景光、远景、主体、前景、标题等），随滚动进度做不同位移和缩放，形成景深与渐显。

### 结构要点

- **外层**：一个占位 div（类名如 `pin-spacer`），高度设为 `400vh`（或 `height: 400vh`），产生长滚动。
- **内层**：一个 `position: fixed` 的容器（类名如 `fgs-timeline`），`top/left: 0`、`width/height: 100vw/100vh`，铺满视口，在滚动过程中保持固定。
- **内容**：在 fixed 容器内放多个子层（如 `fgs-banner` 内再分：背景层、远景层、主体层、前景层、标题层），每层用 `transform: translate3d() scale()` 控制位置与缩放。**水彩古风**下：背景可为淡彩晕染或远山/云气，远景与主体为水彩风格古城、商号、山水等图或 SVG，前景可为淡彩笔触或草木剪影，标题用古典字体、淡赭或青灰色。

### 驱动方式

- 监听 `window` 的 `scroll` 事件（或使用 `IntersectionObserver` 观察 pin-spacer 的可见比例）。
- 用「当前滚动偏移」相对「pin-spacer 从进入视口到离开视口的范围」计算**滚动进度** `t`，使 `t` 从 0 线性变化到 1（例如：`t = (scrollY - start) / (end - start)`，并 clamp 到 [0,1]）。
- 将 `t` 传给各层，用不同系数得到每层的「子进度」或直接用于 transform。

### 每层 transform 示例（可简化）

- **背景/光**：`translate3d(0, t * 20px, 0)` 或 `scale(0.95 + t * 0.1)`，变化慢。
- **远景**：`translate3d(0, t * 40px, 0)`，略快于背景。
- **主体**：`translate3d(0, t * 60px, 0)` 或再加 `scale(0.9 + t * 0.1)`，更明显。
- **前景**：`translate3d(0, t * 80px, 0)`，最快，形成视差。
- **标题/文案**：`translate3d(0, t * 30px, 0)`、`opacity: t`，渐显。

系数（20/40/60/80 等）可按设计微调；若需「先快后慢」可对 `t` 做缓动（如 `t * t`）。

### 实现注意

- 用 `transform: translate3d(...)` 和 `scale()` 做动画，有利于 GPU 加速。
- 图片层可替换为 `<img>` 或 `background-image`，仍用 `style={{ transform: ... }}` 绑定同一套基于 `t` 的计算即可。
- 若项目有路由（如 React Router），可在路由切换时保存/恢复 `window.scrollY` 或滚动到锚点，实现「返回时回到上次滚动位置」的简单效果。

## 实施步骤

### 阶段1：项目基础搭建

1. **安装依赖**
  - 添加 `react-router-dom`
  - 添加 `vis-network` 和 `vis-data`
  - 更新 `package.json`
2. **路由配置** (`src/App.jsx`)
  - 配置 5 个页面：`/`（Home）、`/explore`、`/timeline`、`/ask`、`/about`
  - 挂载全局导航栏，首页保留滚动引入区域
3. **全局导航栏** (`src/components/Navigation.jsx`)（水彩古风）
  - 固定顶部；淡彩底或浅色描边，链接为古典字体、淡青/赭色，hover 可略加深或轻微晕染
  - 包含：首页 | 探索图谱 | 时空叙事 | 智能问答 | 关于
  - 响应式设计

### 阶段2：首页开发 (`src/pages/Home.jsx`)

1. **滚动引入部分**（实现规范见上文「首页滚动引入效果说明（供 AI 复现）」）
  - 提取 `ScrollHero` 组件，**水彩古风**：背景为淡彩晕染或远山/云气，远景与主体为水彩风格古城轮廓、商号剪影或山水，前景可为淡彩笔触或草木；主标题「解码晋商智慧网络」采用古典字体、淡赭或青灰色。
2. **首页内容区**（延续水彩古风）
  - 三大入口卡片：淡彩描边或浅色块、宣纸色底、古典字体；链接：探索知识图谱 → `/explore`、向历史提问 → `/ask`、沿着时间线行走 → `/timeline`
  - 精选故事推荐：轮播 3 个故事片段，卡片与整体淡彩风格一致
  - 数据统计展示：商号数、人物数、关系数（可先占位），数字与标签用淡青/赭色

### 阶段3：知识图谱数据设计 (`src/data/knowledgeGraph.json`)

```json
{
  "nodes": [
    { "id": 1, "label": "日昇昌", "type": "商号", "group": 1 },
    { "id": 2, "label": "雷履泰", "type": "人物", "group": 2 },
    ...
  ],
  "edges": [
    { "from": 1, "to": 2, "label": "创办", "type": "关系" },
    ...
  ]
}
```

- 节点类型：商号、人物、地点、事件
- 关系类型：创办、合作、传承、竞争等
- 初始数据：10-15个节点，20-30条边（用于演示）

### 阶段4：探索图谱页面 (`src/pages/ExploreGraph.jsx`)

1. **图谱可视化组件** (`src/components/GraphVisualization.jsx`)（水彩古风）
  - 使用 `vis-network` 渲染力导向图：节点为淡彩描边或浅色圆点，边为淡青/赭色细线；画布背景可为宣纸色或浅色晕染
  - 节点点击：详情卡片采用淡彩边框、古典字体与留白
  - 节点搜索/筛选：输入框与按钮风格与全站水彩古风一致
  - 图谱缩放、拖拽、布局调整
2. **推荐路线占位** (`src/components/RouteRecommendation.jsx`)
  - 右侧面板「个性化推荐路线」：面板与卡片为淡彩风格，与图谱页统一
  - 占位逻辑：根据选中节点推荐相关节点路径；UI 为路线卡片列表，点击高亮图谱路径；标注「推荐算法占位，后续优化」
3. **页面布局**
  - 左侧：图谱可视化（约 70% 宽）；右侧：节点详情 + 推荐路线（约 30% 宽）；顶部：搜索栏、筛选器（按类型、时间）
4. **页面路由**：`/explore`

### 阶段5：时空叙事页面 (`src/pages/Timeline.jsx`)

1. **时间轴组件**（水彩古风）
  - 垂直时间轴（1800-1911）：淡彩轴线或浅色分隔，关键年份为淡青/赭色小标记，整体柔和如画卷
  - 点击年份显示该年事件；文案与卡片为古典字体、淡彩与留白
2. **故事内容区**
  - 主题故事模式：票号诞生、万里茶路、家族传承；商号传记模式：选择商号查看完整传记与关联事件；版式与配色延续水彩古风
3. **页面路由**：`/timeline`

### 阶段6：智能问答与关于我们

1. **智能问答** (`src/pages/Ask.jsx`)（水彩古风）
  - 问答输入框：淡彩描边、宣纸色底；历史记录与结果展示区版式统一古风，可少量赭色/青灰强调
  - 展示区：模拟或对接问答结果（后续可接大模型/检索 API）；路由：`/ask`
2. **关于我们** (`src/pages/About.jsx`)
  - 项目简介、晋商精神主题说明、致谢或链接；排版与配色为水彩古风（留白、古典字体、淡彩）
  - 路由：`/about`
