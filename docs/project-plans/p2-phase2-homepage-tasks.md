# P2 阶段2 任务说明（首页）

本文档用于指导 **P2 组员**完成首页开发，目标是把首页做成“可滚动叙事 + 古城空间结构解读”的入口页面。

项目主标题统一为：**古城智脑——平遥文化遗产知识图谱与叙事引擎**。

---

## 一、阶段目标

1. 完成首页滚动响应区（Scroll Hero），形成沉浸式进入体验。
2. 完成“空间结构（古城格局）”板块 A.1 的内容呈现。
3. 让用户在首页即可理解：平遥古城“龟城”形制、城墙防御体系、街巷网络逻辑。
4. 首页作为后续 `/explore`、`/timeline`、`/ask` 的内容分发入口。

---

## 二、最终呈现效果（建议落地形态）

### 1. 首屏滚动叙事（Scroll Hero）

用户打开首页后，先进入一个 300vh-400vh 的滚动段。随着滚动：

- 背景层：宣纸肌理 + 淡彩山水/古城轮廓，缓慢位移。
- 中景层：古城平面线稿与局部建筑剪影，速度中等。
- 前景层：标题、引导语、引导箭头，渐显/轻微上移。
- 进度条：页面右下角显示“当前导览进度”。

视觉感受：像在一幅古城长卷中“走入”平遥空间格局。

建议做 6 个画面，叙事节奏为“大画面进入 -> 逐步缩小聚焦 -> 切换到下一画面”。

- 画面 1（0%-18%，全景开场）
  - 画面内容：远景鸟瞰平遥古城全貌，城墙轮廓完整可见，画幅最大。
  - 视觉动作：主画面缓慢放大 1.00 -> 1.08，标题从 0 到 100% 透明度。
  - 文案建议：一城如龟，千年有序。

- 画面 2（18%-34%，全景收束）
  - 画面内容：从全景开始“收镜头”，将焦点落到城墙与城门系统。
  - 视觉动作：主画面由大变小 1.08 -> 0.92，外圈加轻微暗角，中心区域高亮。
  - 过渡方式：缩小到约 0.92 后，叠加线稿层，进入下一画面。

- 画面 3（34%-50%，城墙体系）
  - 画面内容：城门、瓮城、角楼、马面、敌楼、垛口分层浮现。
  - 视觉动作：每类构件依次点亮，标注点做轻微呼吸动效。
  - 文案建议：防御结构不是孤立建筑，而是系统工程。

- 画面 4（50%-66%，城墙到街巷）
  - 画面内容：镜头从城墙内收，转入主街骨架（南大街、东大街、西大街、城隍庙街）。
  - 视觉动作：画面再次由大变小 1.00 -> 0.86，小地图角标出现，提示空间定位。
  - 过渡方式：利用“线网生长动画”替代硬切，视觉上像从城壳走入经络。

- 画面 5（66%-84%，街巷网络）
  - 画面内容：八小街、七十二条蚰蜒巷逐步展开，形成主次分明的路网。
  - 视觉动作：主街线条较粗且亮，小巷线条较细且半透明，节点按功能分类着色。
  - 文案建议：主街定秩序，巷道承日常。

- 画面 6（84%-100%，落版与分流）
  - 画面内容：整体路网缩为中景，右侧出现三入口卡片（Explore / Timeline / Ask）。
  - 视觉动作：主视觉再缩小 0.86 -> 0.78 并固定，入口卡片由下向上淡入。
  - 结束状态：滚动段结束，页面自然进入 A.1 内容区。

补充实现要求（给前端）

- 镜头规则统一：每次“换画面”前都先执行 10%-15% 的缩小段，再做层叠切换。
- 切换类型比例：建议 70% 用缩放+叠化，30% 用线稿覆写，避免频繁硬切。
- 每画面最多 1 个主信息点，避免同屏信息过载。
- 移动端保留 4 画面简化版：全景 -> 城墙 -> 街巷 -> 分流入口。

### 2. 板块 A.1：空间基底与街巷体系

建议分成三张信息卡 + 一张综合可视化画布：

- A.1-1 龟城形制卡：
  - 左侧：龟城总览图（高清实景图 + 手绘线稿叠加）。
  - 右侧：史料摘录与民间溯源要点。
  - 下方：设计寓意标签（如“龟首南向”“护城防御”“聚合街巷”）。

- A.1-2 城墙体系卡：
  - 展示城墙核心数据：城门、瓮城、角楼、马面、敌楼、垛口。
  - 配“知识卡片”点击展开（数量、分布、建筑特征、图像证据）。
  - 在示意图上做“3D 标注点”或“伪 3D 浮层标签”。

- A.1-3 街巷网络卡：
  - 主街（南大街、东大街、西大街、城隍庙街）高亮。
  - 八小街、七十二条蚰蜒巷以层级线网显示。
  - 用关系图表达“主街-巷道-功能节点”的连接结构。

- A.1-4 综合交互画布：
  - 中央为古城平面图。
  - 左侧过滤器：按“城墙构件/街巷层级/历史功能”筛选。
  - 右侧详情面板：点击任一标注后显示图文资料、坐标、关联关系。

### 3. 首页信息组织（从上到下）

1. Scroll Hero（沉浸引入）
2. A.1 空间基底与街巷体系（核心内容）
3. 三个入口卡片（跳转 Explore / Timeline / Ask）
4. 页脚（来源声明、图片版权、史料出处）

## 三、A.1 内容采集与结构化清单

### A.1-1 龟城形制资料

需要收集：

1. 平遥“龟城”形制史料记载（地方志、研究文章、博物馆公开资料）。
2. 民间溯源资料（口述、地方文化文章、权威媒体解释）。
3. 视觉素材：高清实景图、俯瞰图、手绘线稿。

输出要求：

- 一份“龟城形制摘要”（300-500 字）。
- 一张“结构示意图”（可先二维，后续升级为 3D 标注）。
- 3-5 条可视化标签（寓意、布局特征、防御逻辑）。

### A.1-2 城墙体系资料

需要收集：

1. 城墙核心数据与测绘资料。
2. 构件清单及数量、位置、建筑特征：
   - 城门
   - 瓮城
   - 角楼
   - 马面
   - 敌楼
   - 垛口
3. 对应细节图（实景/测绘/剖面示意）。

输出要求：

- 结构化数据表（CSV 或 JSON 均可）。
- 每类构件至少 1 张示意图 + 1 张实景图。
- 可点击知识卡片（字段统一：名称、数量、位置、特征、资料来源）。

### A.1-3 街巷网络资料

需要收集：

1. 主街走向与功能：南大街、东大街、西大街、城隍庙街。
2. 八小街与七十二条蚰蜒巷的路网规律。
3. 平面路网图、航拍图、街巷节点图。

输出要求：

- 一张街巷层级网络图（主街/次街/巷道三级）。
- 一份“主街与小巷功能关联”说明（200-300 字）。
- 一份关系数据（节点+边），供图谱组件调用。

## 四、所需技术（按“必需/增强”分层）

### 1. 必需技术（本阶段应落地）

- React + Vite：页面结构与组件化。
- CSS3 + 原生滚动监听：实现 scroll progress 与视差。
- SVG/Canvas（任选其一）：古城线稿叠加与标注图层。
- 轻量关系图组件（推荐 ECharts Graph 或 vis-network）：街巷网络关系图。
- JSON 数据驱动：A.1 三部分统一用结构化数据渲染。

### 2. 增强技术（有余力再做）

- GSAP ScrollTrigger：更平滑的时间轴滚动编排。
- react-three-fiber + drei：关键构件的 3D 标注与镜头过渡。
- maplibre-gl：把古城底图做成可缩放地图容器。
- Lenis 或同类平滑滚动：提升滚动质感。

### 3. 推荐依赖清单（按需安装）

```bash
npm install echarts
npm install vis-network vis-data
npm install gsap
npm install three @react-three/fiber @react-three/drei
```

说明：不建议一次性全部引入。先保证“必需技术”可运行，再逐步上增强项。

## 五、前端组件拆分建议

建议在 `src/components/home/` 下拆分：

- `ScrollHero.jsx`：滚动引入层。
- `SpatialBaseSection.jsx`：A.1 总容器。
- `TurtleCityCard.jsx`：龟城形制卡。
- `WallSystemCard.jsx`：城墙体系 + 知识卡片。
- `StreetNetworkCard.jsx`：街巷网络图。
- `SpatialCanvas.jsx`：综合标注画布（2D/3D 均可）。

数据建议在 `src/data/home/`：

- `turtleCity.json`
- `wallSystem.json`
- `streetNetwork.json`

## 六、交互细节建议

1. 滚动驱动统一参数：`progress` 范围固定在 `[0, 1]`。
2. 卡片动效：进入视口后淡入 + 轻微上移（避免过度动画）。
3. 知识卡片支持“点击展开/收起”，移动端改为抽屉层。
4. 图上标注支持 hover 与 click 两种状态，避免误触。
5. 所有图文素材保留来源字段，便于后续版权与学术审查。
