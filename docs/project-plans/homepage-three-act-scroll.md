# 首页双幕滚动叙事方案


## 一、为什么改成双幕



推荐叙事链条：

**真实古城进入 → 龟城格局总览（含文献语感）→ 下接龟城形制详解**

这个顺序既能先建立“平遥古城”的空间认知，也能在第二幕用总览 + 文言文把“龟城”概念预埋，用户继续下滚时自然进入 A.1-1 龟城形制（图示、热点、现代阐释），避免首屏内部再开第三幕导致节奏变散。

---

## 二、双幕正式结构

### 第一幕：平遥城头

**定位**：建立开场气质，强调真实古城的空间氛围与中轴秩序；画面以 **分层叠图 + 滚动视差** 完成，不做信息轰炸。

#### 2.1 第一幕图片排布（四层：远景 / 中景 / 左前景 / 右前景）

四张图在同一 **全屏 sticky 视口** 内叠加，自下而上（从后到前）的 **逻辑层级** 建议如下：

| 层级 | 命名 | 在画面中的角色 | 建议内容与构图 | 滚动时的运动倾向（相对最慢层） |
|------|------|----------------|----------------|-------------------------------|
| **L1** | **远景** | 最底层大氛围 | 天空、远处树线、古城屋脊整体延展；可略偏冷、低对比，铺底 | **最慢**：轻微上移或略放大，稳住地平线 |
| **L2** | **中景** | 空间主体与秩序 | 屋顶群、中轴与对称结构；承上启下，连接远景与前景细节 | **中等**：比远景快、比前景慢，形成“走进街巷”的中间带 |
| **L3** | **左前景** | 画面左下（或左侧）近景锚点 | 牌楼、檐角、灯笼、牌匾等 **左侧** 细节；可略冲出安全区增强景深 | **较快**：位移略大（如向左下或略下沉），强化近景压迫感 |
| **L4** | **右前景** | 画面右下（或右侧）近景锚点 | 与左前景 **对称或呼应** 的另一组近景（瓦脊、另一檐角等），避免单侧失衡 | **较快**：与左前景同量级但方向可镜像，形成环抱式构图 |

**排布要点（设计稿层面）**：

1. **构图**：远景铺满或近铺满视口；中景在垂直方向上略收一点（上下留白），突出“城”的体量；左右前景固定在 **下角或两侧**，宽度宜用 `vw` 或固定最大宽限制，避免挡死中轴。
2. **叠放顺序（z-index）**：远景 < 中景 < 左前景 ≈ 右前景（左右同层或微调 1 级均可）。线稿 / 雾面 / 全屏渐变若存在，应单独约定与四层的相对关系（一般压在照片之上、文字之下，或仅压在第二幕）。
3. **素材格式**：若中景或近景为 **带透明通道的 PNG**，需注意与下层混合模式（如 multiply / screen）可能产生发灰、发黑；实现上可单独为某层关闭混合模式或改为 **预合成底图**，以文档中的视觉样稿为准。
4. **与“纯视觉”原则**：首屏滚动段若坚持无字，第一幕仍只保留四层图 + 视差；若后续在第一幕加极弱标题，应固定在中上安全区，不遮挡中轴与左右前景的“门框感”。

**实现备注**：工程上若某一阶段 **暂时去掉中景层**（仅保留远景 + 左右前景），属于素材或性能上的折中，不改变本方案对“四层层级与叙事分工”的定义；恢复中景时应回到上表结构。

#### 2.2 第一幕动效与节奏（摘要）

- 四层 **滚动速度不同**（`transform` 的 `translateY` / `scale` 差分），形成镜头向古城内部推进的感觉。
- 过渡至第二幕时：建议第一幕末尾与线稿 / 地图层 **叠化或位移衔接**（具体用 `opacity` 还是仅用 `transform` 由当期技术约束决定；若约束为“首屏不动态改透明度”，则转场全部用位移与缩放完成）。

**本幕目标**：让用户先被“平遥”的气质与空间秩序吸引，而不是先被信息点打断。

---

### 第二幕：龟城总览 + 文言文介绍（并衔接下方龟城板块）

**定位**：从第一幕的“真实城头”切换到 **格局认知**：用户在此幕应同时获得 **（1）龟城空间总览** 与 **（2）文献层面的“龟城”语感**（文言文摘引），为下方 **「龟城形制」** 板块做好内容与情绪上的铺垫。

#### 2.3 与下方板块的关系（必须写清）

- **第二幕不是孤立结尾**：滚动离开首屏 sticky 区后，**紧接着**就是页面内的 **龟城形制介绍板块**（图示、交互热点、现代汉语阐释等）。
- **叙事分工**：
  - **第二幕（首屏内）**：一屏内完成「总览图 + 短文言文」，强调 **权威感与意象**（为何称龟城、六门象龟等），信息密度 **低、可读、可扫一眼**。
  - **龟城形制板块（首屏下）**：承接同一主题，展开 **形制图解、史料全文或更长译文、交互探索**；避免与第二幕重复堆砌同一段长文。

#### 2.4 第二幕画面与文案结构

**画面**：

- **龟城总览**：使用与项目视觉统一的 **古城总览 / 龟城意象总图**（可与下方板块头图风格一致或略简化），保证用户能一眼识别“整座城的轮廓与六门关系”。
- **动效**：总图可先完整呈现，再 **轻微放大 / 平移** 至中轴或南门（龟头）一带（节奏仍建议保留约 20%～30% 滚动缓冲，避免一晃而过）。
- **底部收束**：首屏最下端用留白、渐变或与下方板块背景色 **同一色相的衔接**，使用户下滚时感觉 **从总览自然“落入”龟城形制模块**，而不是硬切另一皮肤。

**文言文介绍（建议内容形态）**：

- **来源**：宜采用已整理的权威摘引，例如清光绪《平遥县志》等文献中关于 **城池周回、六门、龟形取义** 的原文节选（具体条目以内容组定稿为准）。
- **呈现**：
  - 篇幅：**短篇**，建议控制在 **2～4 句** 或 **一小段**，字体风格与全站古风一致，行距略松，便于扫读。
  - 位置：总览图 **旁侧或下方固定安全区**（不挡核心轮廓）；移动端可改为图下全文流式排列。
  - **现代汉语**：可在龟城形制板块再展开完整译文；第二幕若空间允许，可加 **一行极短译注**（可选，避免首屏字过多）。

**示例（仅作占位，正式稿以史料校对为准）**：

> 城池……形如龟，南门为龟首，北门为龟尾，东西四门为四足。  
> （出处：清光绪《平遥县志》节录，完整版本见下方龟城形制板块。）

**本幕目标**：让用户在离开首屏前已经建立 **“这座城在格局上是一只龟”** 的认知锚点，并带着 **文献语感** 进入下方详解。

## 三、双幕之间的过渡建议

### 3.1 第一幕 → 第二幕

建议从“真实屋顶群”过渡到“龟城总览 + 文言”，避免像换页。

- 第一幕临近结尾时，可略压对比、略统一色温，使地图 / 总图层更易 **叠入**。
- 第二幕总图与文言同时进入时，注意 **总图先稳、文字后稳或同现**，避免字先出图未到造成跳戏。

### 3.2 第二幕 → 下方「龟城形制」板块

- 第二幕结尾：**不另起第三幕**；用底部渐变、背景色连续、或总图位置与下方头图 **视觉对齐**，引导用户继续滚动。
- 下方板块首屏：**直接承接** 龟形图、热点与展开内容；文言文若在第二幕已出现节选，下方可给 **全文、注释与互动**。

## 四、技术实现建议

当前推荐方案：**方案 B（轻量版 Osmo）**

- 使用 `GSAP + ScrollTrigger`
- 暂时不接 `Lenis`
- 用 `sticky + 多层视觉 + 分段状态切换` 实现

**与第一幕四层对应**：

- 第一幕：**远景、中景、左前景、右前景** 各一层 DOM（或 canvas/WebGL 分层），分别绑定不同的滚动进度映射。
- 第二幕：**龟城总览层** + **文言文排版层**（可同一 sticky 内两个子层），总图用 `transform` 做缩放/平移；文言层注意移动端折行与安全区。

**总原则**（可按项目阶段取舍）：

- 优先 `transform`；若产品未限制，可配合 `opacity` 做柔和叠化。
- 少用重度实时滤镜，保证中低端设备可接受。
- 单幕核心主题不超过 1 个：第一幕 = 气质与空间；第二幕 = 龟城总览 + 文献语感。

## 五、素材清单建议

### 第一幕所需

- **远景**：平遥城头 / 屋顶群大景（建议横构图、信息略虚）
- **中景**：中轴与屋脊秩序（可与远景同套拆分或单独渲染）
- **左前景**：左侧近景细节（PNG 透明底需注意混色）
- **右前景**：右侧近景细节（与左前景构图对称或呼应）

### 第二幕所需

- **龟城总览图**：与下方龟城板块风格可衔接的总览 / 示意图
- **文言文**：定稿原文 + 出处标注；如需，另备现代汉语短注一条
- **衔接素材**：底部雾面渐变、与下方板块统一的背景色或纹理

## 六、首页首屏最终目标

首屏双幕完成后，用户应该依次感受到：

1. **第一幕**：这里是平遥，一座真实、有层次、有秩序感的古城（四层画面“走进”古城）。
2. **第二幕**：这座城在格局上是 **龟城**；总览图让我看见全貌，**文言文**让我感到历史与方志的背书。

也就是说，首页首屏不只是一个“好看的开场”，而是 **气质 → 格局 → 文献语感** 的总引子，并 **无缝交给下方龟城形制板块** 做深读与交互。

---

## 七、当前定稿结论

首页首屏滚动叙事现阶段正式采用：

1. **第一幕：平遥城头**（远景 / 中景 / 左前景 / 右前景，四层排布与视差）
2. **第二幕：龟城总览 + 文言文介绍**（直接为下方「龟城形制」板块铺路）

后续执行顺序建议：

1. 定稿四层分切素材与总览图、文言短段
2. 把第二幕末端与下方龟城板块的 **视觉与背景** 衔接做顺
3. 统一滚动节奏与转场（含是否使用动态 `opacity` 的团队约定）
