# 平遥文化网站首页模块项目书（最终版）

## 1. 项目定位与建设目标
本项目首页模块承担全站信息分发与品牌首屏展示职责，核心目标如下：

1. 构建清晰的六大模块入口，降低用户到达各业务板块的路径成本。
2. 建立具有平遥文化辨识度的首屏视觉，强化文化记忆点。
3. 提供稳定的多端访问体验，确保移动端导航可用与页面可读。
4. 通过前端工程化组织页面结构、样式系统与交互逻辑，保障后续可维护性。

## 2. 技术栈与工程结构

### 2.1 技术栈
1. 框架层：React + React Router。
2. 样式层：全局 CSS（集中式样式文件管理）。
3. 交互层：React Hooks（useState/useEffect/useRef）+ 原生滚动与窗口事件。
4. 构建层：Vite。

## 3. 首页模块代码技术分析

### 3.1 页面架构实现
首页采用“固定导航 + 首屏主视觉 + 内容卡片 + 页脚”结构，结构层次清晰：

1. 导航组件独立封装在 Navigation，挂载于应用壳层，保证跨页面一致性。
2. 首页内容由 Home 组件承载，主视觉区采用 scroll-demo 粘性场景布局。
3. 下方使用 page-card-grid 网格卡片呈现核心导览信息。
4. 底部采用 home-footer-mark 呈现品牌与版权标识信息。

对应代码：
1. 首页结构定义：[src/pages/Home.jsx](src/pages/Home.jsx#L53)

3. 页脚标识定义：[src/pages/Home.jsx](src/pages/Home.jsx#L102)

### 3.2 导航栏与六大模块入口实现
项目已通过主导航实现六大入口（首页、历史脉络、营造技艺、晋商文化、民俗非遗、旅游路线）：

1. 六项导航数据采用数组配置，便于维护与扩展。
2. 使用 NavLink 自动管理当前激活态，提升用户定位感。
3. PC 端为横向导航，移动端由汉堡按钮触发下拉菜单。
4. 导航具备滚动显隐策略，减少内容遮挡。

### 3.3 轮播/首屏交互技术实现现状
当前首页并非传统“多图自动轮播”，而是“滚动驱动的一镜到底视差场景”：

1. 通过监听 scroll 与 resize 计算滚动进度 scrollProgress。
2. 将进度映射到多层图形元素的位移、缩放、透明度，形成空间纵深。
3. 进度条通过 scaleX 显示视觉推进状态。
4. 首屏标题与描述文本也随滚动产生位移与透明度变化。

### 3.4 响应式适配实现
响应式策略较完整，关键断点覆盖 900px、640px、1100px、760px：

1. 导航在 900px 以下切换为“按钮 + 下拉菜单”。
2. 页面卡片区在中小屏从三列降为单列，保证阅读舒适度。
3. 首屏主视觉在不同宽度下调整高度、内边距、文本字号与层尺寸。
4. 小屏进一步压缩品牌副标题与操作按钮尺寸。

## 4. 与任务要求对照分析

### 4.1 已满足项

2. 任务要求“六大板块图标入口 + 简短介绍”，当前首页显式图标入口卡片为 3 项，六项入口主要存在于顶部导航。
3. 任务要求“素材来源说明”，当前页脚主要为品牌标识，建议补充独立素材来源文字区。
